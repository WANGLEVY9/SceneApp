# 非功能性需求文档

## 性能
- 接口响应：核心接口（登录、打卡、礼品兑换）在正常负载下后端处理 < 300ms；列表/分页接口 < 500ms。
- 并发：支持至少 200 QPS 峰值（可通过水平扩展实现）。
- 首屏体验：管理端前台首屏 < 2s（本地网络）；用户端页面切换 < 500ms。

## 可用性与可靠性
- 后端异常统一返回标准结构，便于前端识别与兜底提示。
- 关键操作（打卡、兑换）具备幂等/校验逻辑，避免重复提交。
- 日志：按 INFO/ERROR 级别输出，关键链路（登录、打卡、兑换、管理修改）必须有可追踪日志。

## 安全
- 鉴权：JWT 鉴权，管理端要求 `Bearer <token>`；token 过期需返回 401 并提示重新登录。
- 传输：生产建议 HTTPS；内部接口避免明文敏感信息。
- 数据校验：后端对输入做校验与防注入；MyBatis 预编译参数。
- 权限：区分用户端与管理员端接口，敏感操作仅管理员可访问。

## 可维护性
- 模块化：后端分层（controller/service/mapper），公共模块 `scene-common` 复用返回体、异常处理与工具。
- 配置化：分环境配置（dev/local/prod），数据库与端口可配置；前端 API 基址可通过环境变量或方法覆盖。
- 代码规范：DTO/VO/PO 分责，异常统一处理；前端 API 层集中 Axios/Http 工具。

## 可扩展性
- 推荐/排序接口已预留经纬度与排序参数，便于后续接入 LLM/大数据推荐。
- 数据看板可通过增加统计接口接入；管理端仪表盘预留扩展位。
- 礼品兑换与积分规则可配置化（未来外置至表/配置中心）。

## 兼容性与部署
- 后端：JDK 11+，MySQL 5.7/8.0，Maven 构建；可部署于常见云主机/容器。
- 管理端前台：现代浏览器（Chromium/Firefox）；支持反向代理同域部署。
- 用户端前台：HarmonyOS NEXT 设备/模拟器；HttpUtil 基址可按环境覆盖。

## 可测试性
- 接口分组清晰，便于编写 Postman/自动化用例；统一返回体简化断言。
- 数据脚本完整，支持回放测试场景（打卡、礼品、积分）。

## 观测与运维
- 采集：访问日志、错误日志、慢查询、JVM 指标。
- 提供健康检查端点（可在网关/监控中探活）。
